About the Project – Oblig 1

This project is a product showcase website created using BEM methodology, SASS architecture, and layout patterns from Every Layout. I chose to redesign and improve my original submission by implementing a clearer structure, better design decisions, and more modular styling.

I used the BEM methodology consistently across all components, such as .hero, .card, and .button--primary, which made the codebase more readable and scalable. The layout is structured using the Stack layout to manage vertical spacing, allowing me to avoid unnecessary media queries. In the hero section, I followed the Cover pattern by centering the key product message and visuals.

I structured my SASS using a folder-based architecture: base for global styles and design tokens, layout for layout patterns, components for reusable BEM blocks, and utilities for spacing and helper classes. This has improved maintainability and code reuse.

Compared to the original version, this improved version has a stronger visual hierarchy, better accessibility, and a more cohesive design language. I learned how to build scalable, maintainable CSS systems and how to apply design system principles in practice.

I applied logical properties such as margin-inline and margin-block-start instead of traditional directional properties. This improves flexibility and ensures better adaptability across different screen sizes and writing directions.

Responsiveness was achieved without relying heavily on media queries. Instead, I used flexible layout techniques such as flexbox, flex-wrap, and the min() function, allowing elements to adapt naturally to different screen sizes.

Consistency was maintained by using design tokens defined in SASS variables, ensuring consistent colours, spacing, and typography across the project.
