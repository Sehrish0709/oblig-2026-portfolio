Project 3 – Scroll-Driven Storytelling

SDG 14: Life Below Water

Project Overview

This project is an improved version of a previous scroll-driven storytelling assignment created for the course IDG1293 Advanced CSS. The goal of the assignment was to create an interactive and animated web experience based on one of the United Nations Sustainable Development Goals.

I chose SDG 14: Life Below Water because ocean pollution and climate impact on marine ecosystems is an important global issue that affects both nature and people around the world.

The project was redesigned and significantly improved from the original version to create a more immersive storytelling experience. Many of the visual assets and scene compositions were recreated and redesigned in Adobe software to create a more cinematic underwater atmosphere.

Narrative and Scroll Storytelling

The storytelling is built around a visual journey through different ocean depths. The user begins above the surface in a calm beach environment before gradually diving deeper into the ocean.

The project is divided into several scenes:

  • Hero section and beach environment
  • Surface ocean
  • Shallow ocean
  • Mid ocean
  • Deep ocean
  • Ending and call-to-action sections

Each section represents a different emotional stage in the story. The project starts with peaceful visuals and marine life before introducing pollution, damaged ecosystems, and human waste in the deep ocean.

Animations and Interactive Elements

The project uses a combination of CSS animations, GSAP, ScrollTrigger, SVG graphics, and JavaScript interactions.

GSAP and ScrollTrigger were used to create smooth scroll-based animations and reveal effects throughout the project.

  • Floating jellyfish animations
  • Moving dolphin and fish animations
  • Scroll-triggered reveal effects
  • Boat movement and ocean motion
  • Bubble animations
  • Hover effects on pollution items

One of the biggest improvements in the project is the interactive deep-ocean cleanup section. Users can click on pollution items such as anchors, oil spills, garbage, and fish bones to clean the ocean.

When all pollution has been removed, marine life returns to the scene through animated fish, dolphins, and octopus graphics. This interaction makes the user more involved in the storytelling experience instead of only scrolling passively through the website.

Sass Architecture and BEM Methodology

The project was reorganized using Sass partials and BEM naming conventions to improve structure and maintainability.

The CSS architecture was divided into:

  • base/
  • components/
  • utilities/

Separate SCSS partials were created for different sections such as:

  • _surface.scss
  • _shallow.scss
  • _mid.scss
  • _pollution.scss
  • _ending.scss
  • _animations.scss
  • _mediaqueries.scss

BEM methodology was used throughout the project to create scalable and readable class names such as hero__title, shallow-ocean__fish, and deep-ocean__garbagecan.

Accessibility and Responsiveness

Accessibility and responsive design were important considerations during development.

  • Semantic HTML structure
  • Descriptive alt text for SVG images
  • Keyboard navigation using tabindex
  • aria-live regions for dynamic feedback
  • Hover and focus states
  • prefers-reduced-motion media query

Responsive media queries were added to improve scaling, typography, SVG positioning, and spacing across desktop and mobile devices.

Performance Optimisation

SVG graphics were used instead of heavier image formats where possible to improve performance. Simpler animations were created with CSS instead of JavaScript when appropriate, while GSAP was used for more advanced scroll interactions.

ScrollTrigger animations were only activated when needed to improve scroll performance and reduce unnecessary rendering.

Reflection and Improvements

The improved version of the project is significantly more advanced than the original assignment. The original version focused more on separate informational sections, while the new version was redesigned into a connected and immersive storytelling experience.

  • More advanced animations
  • Improved storytelling structure
  • Interactive cleanup mechanics
  • Better responsiveness
  • Accessibility improvements
  • Reorganized Sass architecture
  • More cinematic visual design

The project evolved from a simple informational website into a more emotional and interactive storytelling experience focused on environmental awareness.

Challenges and Problem Solving

One of the biggest challenges during development was positioning SVG elements across different screen sizes while maintaining the visual composition of the scenes.

Another challenge was combining multiple animations, interactions, and responsive layouts without breaking the storytelling flow.

Through testing and iteration, the final result became much more stable, responsive, and visually immersive.
